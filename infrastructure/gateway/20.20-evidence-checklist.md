# 20.20 Evidence Checklist

- `openssl s_client` output for `ui.<domain>:443` showing valid chain and verify code 0.
- `curl https://ui.<domain>/ -I` output without insecure `-k`.
- Certificate metadata output (`subject`, `issuer`, `notBefore/notAfter`) for deployed cert.
- CA trust validation evidence from at least one Windows/macOS/Ubuntu client.
- File permission evidence for `/opt/frontend/certs/*.key` (0600) and cert/bundle files (0644).
