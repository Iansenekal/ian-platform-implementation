# 71.15 Actionable Email Verification Checklist

- Approval email includes explicit Approve and Reject actions.
- Approve link redirects to SSO and enforces MFA before decision commit.
- Reject action requires reason capture in authenticated UI.
- Replayed token is blocked as single-use.
- Forwarded-link decision by unauthorized user is denied.
- Expired-token flow returns safe error and supports authenticated reissue.
- Audit trail includes EMAIL_SENT, ACTION_TOKEN_ISSUED, TOKEN_REDEEMED/TOKEN_EXPIRED/TOKEN_REPLAY_BLOCKED, and APPROVAL_DECISION events.
