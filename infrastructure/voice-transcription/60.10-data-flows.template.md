# 60.10 Data Flows Template

## Flow 1: Manual Upload -> Transcribe -> Store -> Index
1. Upload audio to approved project intake path
2. Record metadata and audit upload event
3. Queue transcription job with classification and ACL context
4. Worker transcribes and writes outputs
5. Record completion and index according to rule

## Flow 2: Review/Redaction Gate
1. Place output in pending-review scope
2. Reviewer approves or rejects with mandatory reason
3. Index only approved output

## Flow 3: Batch Ingestion
1. Scheduled scan of approved scopes
2. Create jobs through same policy path
3. Emit run summary and failure events

## Flow 4: Graph Linkage
1. Create transcript/project/entity/document relationships
2. Enforce ACL-filtered node/edge visibility
