# 40.10 n8n Ports and Boundaries Verification Checklist

- n8n is reachable on 443 only from allowlisted source IPs.
- n8n direct container port 5678 is localhost-only and not reachable from LAN.
- Reverse proxy enforces internal TLS and expected Host header routing.
- Outbound policy is deny-by-default and only approved destinations are reachable.
- n8n has no Docker socket or host-level privileged API access.
- Integration credentials are scoped and stored via secrets standard.
