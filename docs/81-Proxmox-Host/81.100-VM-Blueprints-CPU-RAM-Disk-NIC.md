## Purpose
Define deterministic VM hardware/NIC blueprints for:
- `AI-DATA01`
- `AI-FRONTEND01`
- `DNS01` (optional)

## Scope
VM hardware and NIC design only. OS/application configuration is covered in service deployment runbooks.

## Golden Rules
- Static IPs for all platform VMs.
- One NIC per VM on `vmbr0` (VLAN tag only when VLAN mode enabled).
- QEMU Guest Agent is mandatory.
- Use VirtIO SCSI for disks and VirtIO NIC model.
- Use UEFI (`OVMF`) unless legacy constraints exist.
- VM disks on primary datastore (`vm-disks` or `zfs-vm`), backups on `backup-store`.

## Lab Defaults
- LAN: `10.10.5.0/24`
- Gateway: `10.10.5.1`
- DNS: `10.10.5.160` (or `DNS01` if enabled)
- Domain: `lab.local`

## Naming
- `AI-DATA01` (`ai-data01.lab.local`)
- `AI-FRONTEND01` (`ai-frontend01.lab.local`)
- `DNS01` (`dns01.lab.local`, optional)

## Blueprint Summary
| VM | Role | vCPU | RAM | System Disk | Data Disk(s) | NIC |
|---|---|---:|---:|---:|---:|---|
| AI-DATA01 | Backend/data services | 8 | 32 GB | 64 GB | 500 GB to 2 TB | 1x VirtIO on vmbr0 |
| AI-FRONTEND01 | UI/ingress | 4 | 16 GB | 64 GB | 100 to 250 GB | 1x VirtIO on vmbr0 |
| DNS01 (optional) | Internal DNS | 1-2 | 2 to 4 GB | 16 to 32 GB | N/A | 1x VirtIO on vmbr0 |

## Proxmox Hardware Standards
- Machine type: `q35`
- BIOS: `OVMF`
- SCSI controller: `VirtIO SCSI single`
- NIC model: `VirtIO`
- QEMU Guest Agent: enabled

## QEMU Guest Agent (Mandatory)
Must be true for all Ubuntu VMs:
- Proxmox UI shows guest IP.
- Proxmox options show `QEMU Guest Agent = Enabled`.
- Guest service runs in OS.

- Datastore selection confirmed.
