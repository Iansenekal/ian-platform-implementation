# 81.20 Proxmox Host Install Step-by-Step

Document ID: `docs/81-Proxmox-Host/81.20-Proxmox-Install-StepByStep`  
Version: `v1.0`  
Date: `2026-02-10`  
Owner: `Infrastructure / Platform Ops`

## Purpose
Perform a clean Proxmox VE install on the host with correct disk selection, static management networking, and verified first login.

## Scope
Proxmox installation only (no VM creation).

## Safety Notes
- Selected install disk will be wiped.
- Use wired Ethernet only.
- Record hostname/IP/DNS/gateway/disk choices for deterministic rebuilds.

## Required Inputs
| Item | Value |
|---|---|
| Proxmox hostname | `pve01.lab.local` (example) |
| Proxmox management IP | `10.10.5.xxx/24` |
| Gateway | `10.10.5.1` |
| DNS server | `10.10.5.160` |
| Search domain | `lab.local` |
| Timezone | `Africa/Johannesburg` |
| Install disk | Target NVMe/SSD |
| Admin email | Ops alert email |

## Procedure

### 1) Create install media
- Download stable Proxmox VE ISO from official source.
- Write ISO to USB (Rufus/Etcher).
- Label USB (`PROXMOX_INSTALL`).

Verification:
- USB boots in UEFI boot menu.

### 2) Boot installer
- Insert USB.
- Boot host and open boot menu (vendor-specific key).
- Select UEFI USB entry.
- Choose `Install Proxmox VE`.

Verification:
- Installer welcome screen visible.

### 3) Installation wizard

#### EULA
- Accept EULA.

#### Target disk (critical)
- Select intended install disk.
- Verify capacity/identity.
- Use advanced options only if planned (ZFS/etc.).
- Default lab recommendation: `EXT4 + LVM-thin`.

Evidence:
- Capture disk-selection screenshot.

#### Locale/time/keyboard
- Timezone must be `Africa/Johannesburg`.

#### Admin credentials
- Set strong root password (vault-managed).
- Enter alert email.

#### Management networking
- Select management NIC.
- Set hostname and static IP.
- Set gateway and DNS.

Evidence:
- Capture network-config screenshot.

#### Confirm + install
- Re-check disk and network values.
- Start install.
- Remove USB at prompt.
- Reboot.

### 4) First boot verification (host shell)
Run:

```bash
ip a
ip r
cat /etc/hosts
cat /etc/resolv.conf
ping -c 2 10.10.5.1
ping -c 2 10.10.5.160
```

Pass criteria:
- Static management IP is present.
- Default route points to gateway.
- Hostname mapping exists.
- DNS nameserver correct.
- Gateway and DNS reachable.

### 5) First web login verification
- Open: `https://<Proxmox-IP>:8006`
- Accept self-signed warning (lab default).
- Login as `root`.
- Confirm dashboard/node/networking display expected values.

### 6) Stop-here gate
Do **not** proceed to VM creation until all pass:
- Host network checks pass.
- Web UI reachable from admin workstation.
- Hostname correct.
- No unexpected first-boot errors.

## Mandatory Evidence
- Disk selection screenshot.
- Network configuration screenshot.
- First-login dashboard screenshot.
- Saved command-output evidence.
